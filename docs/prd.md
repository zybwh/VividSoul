# **VividSoul - AI 驱动的开放式 3D 桌面伴侣**  
**Product Requirements Document (PRD)**

**版本**：v1.0  
**日期**：2026年4月4日  
**作者**：Grok（基于用户需求总结）  
**状态**：草稿

## 1. 产品概述

### 1.1 产品定位
**VividSoul** 是一款**轻量级、前后端分离**的 **AI 驱动 3D 桌面伴侣（桌宠）**。

- **核心理念**：不做模型、不做内容，只做**“灵魂”** —— 通过服务端强 LLM 赋予任意 3D 模型（主要是 VRM / glTF）智能理解能力和动态行为。
- **差异化**：完全开放的模型生态 + LLM 实时指令驱动动作/表情/物理 + 极致轻量桌面体验。
- **目标**：让用户只需拖入一个 .vrm 文件，就能拥有一个会思考、会互动、能聊骚、能根据指令“活”起来的 3D 桌面伴侣。

### 1.2 核心价值主张
- **模型零瓶颈**：用户/社区可随意导入自己的 3D 模型（包括大量 NSFW/裸体模型）。
- **智能而非预设**：LLM 理解自然语言指令，输出结构化 JSON 实时驱动模型动作、表情、物理反应。
- **成人向友好**：视觉尺度由模型本身决定，LLM 只负责智能回应和轻度行为引导，无任何审查。
- **轻量极致**：前端轻到能常驻桌面，后端灵活对接任意 LLM（云或自建）。

## 2. 目标用户

- **核心用户**：喜欢桌面伴侣、VTuber、AI 女友/男友、NSFW 角色扮演的用户。
- **次要用户**：想自己做/分享 3D 模型的创作者、希望拥有私人 AI 伴侣的开发者/程序员。
- **痛点**：现有桌宠要么模型封闭、要么 AI 弱、要么技术栈重（Unity）、要么不能完美漂浮在桌面。

## 3. 核心功能需求

### 3.1 桌面体验（桌宠模式）
- 透明无边框 Overlay（Windows + macOS 均支持）
- 点击穿透（点击模型外区域穿透到桌面）
- Always-on-Top + 可拖拽 + 滚轮缩放 + 边缘吸附
- 全局热键呼出/隐藏/唤醒对话
- 最小化到任务栏/系统托盘，常驻后台

### 3.2 模型生态（最重要）
- 支持 **VRM 0.x / 1.0**（优先）及 glTF 扩展模型
- 一键拖拽导入本地 .vrm 文件
- 支持 SpringBone 物理、BlendShape 表情、VRMA 动画
- 模型配置文件（JSON）允许映射：
  - 情绪 → 表情
  - 关键词/指令 → 动作组
  - 自定义物理参数（衣服抖动、呼吸等）
- 社区模型仓库友好（后续可做一键导入/分享）

### 3.3 AI 互动核心（我们的竞争力）
- **前后端彻底分离**：前端只负责渲染 + 交互，后端为独立 LLM 服务
- LLM 输出**结构化 JSON**（emotion、action、intensity、pose、text 等）
- 前端实时解析 JSON 并驱动模型：
  - 表情切换
  - 动作/动画触发
  - 物理强度调节
  - 头部/眼睛跟随（LookAt）
  - 唇同步（TTS）
- 支持语音输入（STT）+ 语音输出（TTS）
- 可选长期记忆、角色设定、上下文理解

### 3.4 成人向支持
- 对 NSFW 模型零限制（裸体、衣服可脱层、敏感部位物理）
- LLM 可使用轻度成人向 prompt（“用挑逗/温柔语气回应”）
- 鼠标悬停/点击特定部位可触发专属反应（可选）

### 3.5 技术非功能需求
- **前端**：Tauri（首选）或 Electron（备选），强力推荐 Svelte / Three.js + three-vrm 栈
- **后端**：Node.js / Python / Go 均可，WebSocket 通信
- **LLM**：支持 OpenAI、Claude、Grok、Gemini、Ollama、本地模型等任意后端
- **体积**：安装包 < 50MB（目标）
- **资源占用**：后台 < 300MB 内存
- **跨平台**：Windows + macOS（Linux 后续）

## 4. 成功指标（MVP 阶段）
- 成功导入任意 NSFW VRM 模型并正常物理互动
- LLM 能通过自然语言指令让模型做出对应智能动作
- 稳定运行透明桌宠模式（点击穿透 + 拖拽）
- 启动速度 < 3 秒

## 5. 后续扩展方向（Phase 2+）
- 模型市场 / 社区分享系统
- 多角色同时显示
- 屏幕感知 / 剪贴板读取 / 自动化工具调用
- 完全本地化版本（集成本地 LLM）
- Steam / itch.io 上架
